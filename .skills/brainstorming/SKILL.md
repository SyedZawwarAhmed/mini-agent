# Brainstorming

Help turn rough ideas into a clearer design through focused collaboration.

## When to use this skill

- The user is shaping a feature or change before implementation
- The requirements are incomplete or still evolving
- The user wants approaches, trade-offs, or design guidance
- The user asks how to structure or think through a problem before coding

## How to respond

1. Understand the goal first
   - Clarify the problem, constraints, and success criteria
   - Ask only the most important follow-up questions

2. Explore alternatives
   - Present 2-3 reasonable approaches
   - Lead with the recommended option
   - Explain trade-offs concisely

3. Turn the idea into a usable design
   - Describe architecture, components, data flow, and failure modes
   - Keep the explanation practical and implementation-oriented

## Principles

- Avoid jumping into code before the problem is clear
- Prefer clear trade-offs over a single vague recommendation
- Keep the conversation focused and incremental
- Remove unnecessary complexity aggressively
